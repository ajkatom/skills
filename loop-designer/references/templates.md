# Output File Templates

Fill every `<placeholder>` with a concrete, project-specific value. Never ship a
placeholder. Only create the files for loops the project actually uses.

---

## `LOOPS.md` (always — the hub)

```markdown
# Loops for <Project Name>

Loop design generated by the loop-designer skill on <date>.
These commands are ready to paste — nothing runs automatically.

## Chosen loops & why
| Loop | Used? | Why |
|------|-------|-----|
| Turn-based (verification) | yes/no | <reason> |
| Goal-based (/goal)        | yes/no | <reason> |
| Time-based (/loop,/schedule) | yes/no | <reason> |
| Proactive (unattended)    | yes/no | <reason> |

## Launch commands

### Goal-based
```
/goal <criteria in one line>, stop after <N> tries
```
(criteria detailed in goal.md)

### Time-based
```
/loop <interval> <what to do each cycle>
```
or, for the cloud:
```
/schedule <cadence>: <what to do each run>
```

### Proactive
```
/schedule <cadence>: <trigger check>. /goal: <per-run stop condition>.
  <auto-mode / model-routing / parallelism notes>
```

## Notes
- Verification: <inline in CLAUDE.md | verify-<task>/SKILL.md>
- Cost guardrails: <model routing, run frequency, pilot slice>
```

---

## `goal.md` (goal-based only)

```markdown
# Goal: <one-line objective>

## Success metric
<metric> must reach <threshold>.

## Evaluator
Command that produces the number:
`<command>`
Interpretation: pass when `<condition>`.

## Stop conditions
- Goal met (metric ≥ threshold), OR
- <N> attempts reached.

## Ready command
`/goal <criteria>, stop after <N> tries`
```

---

## `CLAUDE.md` — Verification section (trivial checks) OR pointer (heavy checks)

Trivial (inline):
```markdown
## Verification
Before considering any change done, run:
`<single deterministic command>`
Pass = <condition>.
```

Heavy (pointer only):
```markdown
## Verification
Self-check every change with the `verify-<task>` skill
(.claude/skills/verify-<task>/SKILL.md) — do not report done until it passes.
```

---

## `.claude/skills/verify-<task>/SKILL.md` (heavy turn-based verification)

```markdown
---
name: verify-<task>
description: Use to verify <task> is actually done — <what it checks>. Run before
  reporting any <task> change complete.
---

# Verify <task>

Run these checks end-to-end. Report done only when all pass.

1. <start server / build step>
2. <interact: open <route>, click <element>, take screenshot>
3. <quantitative gate: zero new console errors, <metric> ≥ <threshold>>
4. <deterministic command> exits 0

If any check fails, fix and re-run — do not stop at "good enough."
```
