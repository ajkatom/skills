# Enterprise assurance tier — the ONE non-stdlib dependency in dark-factory.
#
# Used exclusively by scripts/df_custody.py for ed25519 split-custody
# signing/verification (K-of-N approver sign-off). No other module in this
# codebase imports `cryptography`; the rest of dark-factory (cooperative,
# standard, hardened tiers) runs on the Python standard library alone.
#
# Install only if you are running (or testing) the enterprise tier:
#   pip install -r dark-factory/requirements-enterprise.txt
#

# needs, capped below the next major so a future breaking release can't
# silently land on an unpinned install. Currently tested against 49.0.0.

# the project adopts a hash-locked install flow (see
